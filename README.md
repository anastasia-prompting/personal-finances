# Личный финансовый AI-аудитор — Android MVP

Нативный Android-проект на Java/Kotlin-compatible Android SDK без серверной части.

## Что реализовано в v1

- Нативный Android-проект с `MainActivity` и локальной SQLite-базой.
- Две темы интерфейса: светлая и тёмная.
- Пользовательское соглашение при первом запуске и повторный просмотр из главного экрана.
- Быстрый ввод операций:
  - Доход
  - Расход
  - Перевод в фонд
  - Расход из фонда
  - Платёж по кредиту
  - Увеличение долга
- Счета: основная карта, наличные, кредитная карта.
- Фонды: подушка, жильё, развитие, удовольствия и др. (без Ozon/WB — они не фонды).
- Маркетплейс-кошельки (платёжные контейнеры): Ozon-кошелёк, WB-кошелёк — отдельные счёта, не фонды.
- Категории доходов и расходов.
- Кредитная логика: долг растёт через «Увеличение долга» и уменьшается через «Платёж по кредиту».
- Главный экран с балансами, фондами, долгом, доходами и расходами за **текущий календарный месяц** (перевод на маркетплейс в расходы месяца не входит; покупка с кошелька Ozon/WB — обычный расход и в расходы месяца входит).
- Журнал последних операций.
- Удаление операций.
- Экспорт CSV в app-specific external storage (в файл попадает **полная** история операций; на главном экране по-прежнему показываются только последние 20).

## Как открыть

Открыть **корень** этого репозитория (в `settings.gradle` имя проекта — `FinancialAIAuditor`) в Android Studio.

## Как собрать APK

В Android Studio:

1. File → Open → выбрать папку с проектом (корень, где лежат `settings.gradle` и `app/`).
2. Дождаться Gradle Sync.
3. Build → Build Bundle(s) / APK(s) → Build APK(s).
4. APK будет в `app/build/outputs/apk/debug/app-debug.apk`.

### Gradle Wrapper в репозитории

В корне может не быть каталога `gradle/wrapper` и скриптов `gradlew` / `gradlew.bat` — тогда **командная строка** `./gradlew assembleDebug` заработает только после появления обёртки. Сделать так:

- **Через Android Studio** (удобнее всего): открыть проект и выполнить успешный Gradle Sync; в настройках *File → Settings → Build, Execution, Deployment → Build Tools → Gradle* обычно выбирают **Gradle Wrapper**. После первого успешного синхронизированного билда Studio часто создаёт wrapper в проекте; если нет — см. ниже.
- **Через установленный Gradle** на машине: в корне проекта выполнить `gradle wrapper` (при необходимости укажите версию Gradle, совместимую с Android Gradle Plugin 8.6.x — см. [таблицу совместимости](https://developer.android.com/build/releases/gradle-plugin#updating-gradle)). Появятся `gradlew`, `gradlew.bat` и `gradle/wrapper/`.
- **Вручную не копируйте** файлы wrapper из сомнительных источников — лучше сгенерировать командой выше или доверить создание Android Studio.

После появления wrapper:

```bash
gradlew.bat assembleDebug
```
(в PowerShell/CMD из корня проекта; на Linux/macOS — `./gradlew assembleDebug`).

## Важно

На машинах без Android SDK и без локального Gradle автоматическую сборку APK в этой среде выполнить нельзя; используйте Android Studio на своём ПК.

## Пользовательское соглашение

Встроено в приложение. Соглашение фиксирует, что приложение предназначено для личного учёта и ориентировочного планирования, не является финансовой, инвестиционной, налоговой или юридической консультацией, а данные хранятся локально.

## Итерация: финансовые контейнеры и маркетплейс-кошельки

В этой версии уточнена модель учёта:

- пользовательская терминология в интерфейсе смещена от «счетов» к картам, наличным, фондам и маркетплейс-кошелькам;
- Ozon и Wildberries перенесены в блок карт/кошельков, а не фондов;
- добавлен тип операции «Перевод на маркетплейс»;
- источник перевода на маркетплейс: активная дебетовая или кредитная карта;
- наличные не используются как источник перевода на маркетплейс;
- перевод с дебетовой карты уменьшает карту и увеличивает маркетплейс-кошелёк;
- перевод с кредитной карты увеличивает маркетплейс-кошелёк и долг, но не считается расходом месяца;
- покупка с маркетплейса оформляется как обычный расход с источником Ozon/WB-кошелёк и категорией покупки;
- «Продукты и Быт» остаётся операционным фондом-ориентиром и не предлагается как копилка для перевода, чтобы не создавать двойной учёт;
- добавлена первичная настройка после пользовательского соглашения: карты, наличные, маркетплейс-кошельки, фонды;
- до первого релиза миграции базы данных остаются разрушительными: при обновлении тестовой версии лучше удалить приложение или очистить данные в эмуляторе.
